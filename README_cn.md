## 概述

本仓库包含Wei 开发的多模型交叉验证辅助决策技能，用于Openclaw 和 Claude Code场景。

---

## 核心优势

### 🔍 1. 通过交叉验证减少幻觉
多个独立模型回答同一查询。冲突会被显式检测和解决，相比单一模型系统显著降低幻觉风险。

### 🌐 2. 多源情报
不同模型利用不同的训练分布、检索系统和推理模式——提高覆盖范围并减少盲点。

### 🧠 3. 更高置信度的 AI 回答
当多个模型趋于一致时，置信度增加；当它们出现分歧时，不确定性会被呈现而非隐藏。

### ⚙️ 4. 可用于生产环境的 AI 应用
专为以下场景集成设计：
- AI 智能体
- 研究副驾驶
- 企业知识系统
- LLM 编排管道

## 🧠 wei-cross-research

> 多模型 AI 研究系统，生成**高置信度、基于证据的回答**。

### 功能说明

`wei-cross-research` 是一个 AI 编排技能：

1. 将同一查询并行发送给多个 LLM
2. 收集独立回答
3. 比较推理路径和事实声明
4. 使用评判模型合成最终基于证据的回答
5. 显式呈现分歧和不确定性

这创建了一个**交叉验证的 AI 研究管道**，而不仅仅是单一模型的回答。

---

### 何时使用

当你需要以下能力时，请使用 `wei-cross-research`：

### ✅ 高置信度回答
- 投资/商业分析
- 技术决策
- 研究综合

### ✅ 多视角推理
- 复杂或模糊的问题
- 存在冲突观点的主题
- 信息不完整的领域

### ✅ 降低幻觉风险
- 当正确性比速度更重要时
- 当单一模型回答可能不可靠时

---

### 为什么有效

传统 LLM：
> 单一模型 → 单一回答 → 隐藏不确定性

Wei Cross-Research：
> 多个模型 → 竞争性回答 → 显式验证 → 基于证据的合成

这种设计改进了：
- 事实可靠性
- 推理稳健性
- 不确定性透明度

---

### 成本与性能权衡

> ⚠️ 重要提示：这是一个**多模型系统**，不是单一 API 调用。

典型使用成本：
- 相比单一模型查询约 2-3 倍 token
- 由于并行执行导致延迟更高

何时使用：
- 准确性 > 成本
- 可靠性很重要

何时避免：
- 简单问答
- 低风险查询

---

### 环境要求

- [Bun](https://bun.sh) >= 1.0
- 支持的 LLM 提供商 API 密钥（例如 OpenRouter、Bailian 等）

---

### 使用方法

#### Claude Code / OpenClaw

```bash
use wei-cross-research

```

高级用法：

```
use wei-cross-research

query="美联储2026年会降息吗？"
queryType="financial"

```

## 🎯 wei-devils-advocate

> 通过生成反驳论点进行对抗性审查，压力测试想法。

### 功能说明

`wei-devils-advocate` 是一个对抗性推理技能：

1. 接收你的论点或想法作为输入
2. 查询多个批评模型以生成最强反驳论点
3. 探索现实的失败场景
4. 使用评判模型评估可存活性
5. 提供结构化裁决及建议

这强制执行**"默认怀疑，赢得信心"**——强大的想法在攻击中存活，弱小的想法不能。

---

### 何时使用

当你需要以下能力时，请使用 `wei-devils-advocate`：

### ✅ 通过挑战进行验证
- 产品和创业验证
- 投资/交易风险分析
- 策略压力测试

### ✅ 识别隐藏风险
- 发现未受挑战的假设
- 暴露被低估的风险
- 呈现确认偏误

### ✅ 高风险决策
- 当"是"来得太容易时
- 当你需要在压力下的真相时
- 在投入重大资源之前

---

### 为什么有效

传统验证：
> 寻求确认 → 找到支持证据 → 虚假信心

Wei Devil's Advocate：
> 寻求反驳 → 生成反驳论点 → 对抗性评估 → 赢得信心

这种设计暴露了：
- 隐藏假设
- 盲点和偏见
- 失败模式（在发生之前）

---

### 成本与性能权衡

> ⚠️ 重要提示：使用多个模型（相比单一查询 2-4 倍成本）。

典型使用成本：
- 反驳生成约 2-3 倍 token
- 评判评估额外 token

何时使用：
- 风险很高时
- 糟糕决策有实际后果时
- 你想尽早"扼杀"糟糕想法时

何时避免：
- 你只是想要验证时
- 低风险的头脑风暴
- 速度 > 彻底性

---

### 环境要求

- [Bun](https://bun.sh) >= 1.0
- 支持的 LLM 提供商 API 密钥（OpenRouter、Bailian/DashScope）

---

### 使用方法

#### Claude Code / OpenClaw

```bash
use wei-devils-advocate

query="我们应该在 2026 年投资 AI 创业公司吗？"
```

带显式配置：

```
use wei-devils-advocate

query="微服务架构是否适合我们的创业公司？"
queryType="technical"
mode="attack"
```

### 输出结构

1. **论点** — 重新陈述的输入声明
2. **隐藏假设** — 必须成立的隐含信念
3. **反驳论点** — 来自每个模型的最强挑战
4. **失败场景** — 这可能失败的现实方式
5. **可存活性** — 对想法稳健性的评估
6. **裁决** — `fatal` / `risky` / `robust`
7. **建议** — `kill` / `pivot` / `proceed`
