# Wei Skills

## 什么是 Wei Skills？

Wei Skills 不仅仅是一个工具包。

它是一个**AI 系统的推理层**，将不可靠的单模型输出转化为**高置信度、决策级智能**。

与其向一个模型要答案，Wei Skills 选择：

- 让多个模型独立思考
- 迫使它们相互质疑、挑战和验证
- 只有在冲突解决后才合成最终答案

> **AI 不应该生成答案——它应该赢得答案。**

---

## 问题所在

大多数 LLM 应用依赖单一模型。

> ❌ 一个模型 → 一个答案 → 隐藏的不确定性

这导致：
- 幻觉
- 过度自信
- 脆弱推理

---

## Wei 方法论

Wei Skills 引入了一种新范式：

> ✅ 多个模型 → 竞争性推理 → 显式验证 → 基于证据的合成

这将 AI 从：

- **答案生成** → **真理探寻过程**
- **黑盒输出** → **透明推理系统**

---

## 为什么重要

Wei Skills 专为**出错成本高昂**的场景而构建：

- 投资决策
- 技术架构
- 产品与战略验证
- 研究综合

> 当准确性至关重要时，**共识胜过自信**。

### 🧠 通过收敛获得信心
共识增加信心。分歧揭示不确定性。

## 它真正做什么

`wei-cross-research` 不仅仅是多模型查询。

它是一个**共识引擎**。

不满足于第一个答案，它选择：

1. 强制多个模型独立思考
2. 比较推理和事实声明
3. 识别冲突和不一致
4. 通过评判模型解决分歧
5. 产生**收敛的、基于证据的答案**

> 这不是更快的 AI。
> 这是**更值得信赖的 AI**。

# 🎯 wei-devils-advocate

> **在现实的打击之前，先打破看似真实的事物。**
> 通过多模型对抗性推理对想法进行压力测试。

🚀 试用：https://www.bigbigai.com/agent/devils-advocate

## 它真正做什么

`wei-devils-advocate` 不仅仅是生成反驳论点。

它是一个**失败引擎**。

不问"这是个好主意吗？"，而是问：

它是如何失败的？

它通过以下方式工作：

1. 接受你的论点或想法
2. 从多个模型生成最强的反驳论点
3. 暴露隐藏假设
4. 模拟现实的失败场景
5. 评估想法是否在攻击中存活

> 弱小的想法崩溃。
> 强大的想法赢得信心。

---

## 何时使用

在需要以下能力时使用 `wei-devils-advocate`：

### ⚠️ 高风险决策
- 投资/交易
- 产品战略
- 架构选择

### 🧠 偏见检测
- 当你已经"喜欢"这个想法时
- 当某件事感觉显然正确时

### 🔍 真理高于验证
- 在投入资源之前
- 在不可逆决策之前

---

## 心智模型

传统思维：

> 想法 → 验证 → 确认

Wei 魔鬼代言人：

> 想法 → 攻击 → 打破 → 存活 → 决定

## 输出理念
- 假设被明确化
- 风险被迫浮出水面
- 信心必须赢得

> 如果它存活下来，你可以更信任它。
>  如果它崩溃，你节省了实际成本。

## 你得到什么
- 隐藏假设
- 强有力的反驳论点
- 失败场景
- 可存活性裁决：
    - fatal（致命）
    - risky（有风险）
    - robust（稳健）

---

## 成本与性能权衡

> ⚠️ 重要提示：使用多个模型（相比单一查询成本 4-6 倍）。

典型使用成本：
- 反驳生成约 2-3 倍 token
- 评判评估额外 token

何时使用：
- 赌注很高
- 糟糕决策有实际后果
- 你想尽早"扼杀"糟糕想法

避免使用：
- 你只是想得到验证
- 低风险的头脑风暴
- 速度 > 彻底性

## 使用方法

### Claude Code / OpenClaw

```bash
use wei-devils-advocate

query="我们应该在 2026 年投资 AI 创业公司吗？"
```

带显式配置：

```
use wei-devils-advocate

query="微服务架构是否适合我们的创业公司？"
queryType="technical"
mode="attack"
```

### 输出结构

1. **论点** — 重新陈述的输入声明
2. **隐藏假设** — 必须成立的隐含信念
3. **反驳论点** — 来自每个模型的最强挑战
4. **失败场景** — 这可能失败的现实方式
5. **可存活性** — 对想法稳健性的评估
6. **裁决** — `fatal` / `risky` / `robust`
7. **建议** — `kill` / `pivot` / `proceed`
